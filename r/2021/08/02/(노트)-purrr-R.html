map, map_* : list(a=1:5, b:1:30) $\to$ list(fun(a),fun(b))

- map, map_* 의 특징

1 C로 구현되어 있어 빠르다. 하지만 가독성이 희생되었다. (읽기 어렵다)

2 두번째 인수는 적용할 함수를 나타내며 .f로 되어있는데 이는 공식, 문자형벡터, 정수형벡터등을 지정할 수 있다.

3 map_* 은 .f가 호출이 될 때마다 추가인수를 ... 사용하여 전달할 수 있다. 여기에서 ... 은 길이가 정해지지 않은 입력.

4 map, map_* 의 첫번째 인수는 두번째 인수인 함수에 전달할 첫 인자이다. (이게 중요하다 $\star\star\star$)

사실 보통사람들은 map(data, function)의 형태로 외우는데 이는 엄밀하게 말하면 사실이 아니다. map함수의 역할은 단지 $${\tt map}\big([x1,x2,x3],f\big)=\big[f(x1),f(x2),f(x3)\big]$$ 이거나 $${\tt map}\Big(\big[[x1,y1],[x2,y2],[x3,y3]\big],f\Big)=\Big[f(x1,y1),f(x2,y2),f(x3,y3)\Big]$$ 일 뿐이다. 대부분의 함수 $f$가 첫번쨰 인자로 data를 받기 때문에 이러한 오해가 생기는것.

ex1: map, map_* 차이

- map_dbl(df,mean) = list (mean(df[[1]]),mean(df[[2]]), ... , mean(df[[n]]))

map_dbl(df, mean) %>% print()
map_dbl(df, median) %>% print()
map_dbl(df, sd) %>% print()

            a             b             c             d 
 0.3198993385 -0.2495560154 -0.3744838885  0.0009570046 
         a          b          c          d 
 0.4015112 -0.6895933 -0.0696422  0.5150910 
        a         b         c         d 
0.8986917 1.1163977 1.1914158 0.9909110

- 결과는 벡터로 나온다. (벡터는 길이가 4이고 각 원소마다 a,b,c,d 라는 이름이 있음)

- 결과가 벡터이므로 아래와 같은 연산이 가능함

print(map_dbl(df, mean) + 1 )

        a         b         c         d 
1.3198993 0.7504440 0.6255161 1.0009570

- 아래와 같이 각 원소에 이름이 있는 벡터를 선언할 수 있음.

c(a=1.1,b=2.2,c=3.3,d=4.4) %>% print()

  a   b   c   d 
1.1 2.2 3.3 4.4
