(노트) einsum 질문대응

Mar 22, 2022 • 신록예찬 • 5 min read

데이터과학

  • (2,3) @ (3,4) 행렬곱
  • (3,) @ (3,) -> None (무차원, 무축, 축이0개, scalar)
  • (3,) @ (3,) -> (3,3)
  • 아인슈타인법은 텐서간의 연산 일반화 버전임
  • np.einsum은 다양한 텐서연산으로 확장할 수 있음. 그런데 @는 2차원 매트릭스만 특화된 연산자임 
import numpy as np

(2,3) @ (3,4) 행렬곱 

A = np.array(range(2*3)).reshape(2,3)
B = np.array(range(3*4)).reshape(3,4)

A는 (2,3) martix -> A매트릭스의 각 원소는 $a_{ij}$로 볼 수 있고 i=1,2 and j=1,2,3

B는 (3,3) matrix -> B매트릭스의 각 원소는 $b_{jk}$로 볼 수 있고, j=1,2,3 and k=1,2,3,4

A @ B

array([[20, 23, 26, 29],
       [56, 68, 80, 92]])

(1,1)의 원소는 $a_{11}b_{11}+a_{12}b_{21}+a_{13}b_{31}$ 으로 볼 수 있음

A[0,0]*B[0,0] + A[0,1]*B[1,0] + A[0,2]*B[2,0]

20

유사한 논리로 (i,k)의 원소는 아래와 같이 볼 수 있음

$\sum_{j=1}^{n} a_{ij}b_{jk}$

(재해석)

가능한 모든 $a_{ij}$, $b_{jk}$에 대하여 $\sum_{j=1}^{n} a_{ij}b_{jk}$와 같은 연산을 정의한다면 결과는 $c_{ik}$로 나옴 = 최종차원 ik

마치 $a_{ij}, b_{jk} \to c_{ik}$ 로 가는 변환이 이루어진것 같으므로 ij,jk -> ik이라는 약어로 선형변환을 해석할 수 있음

np.einsum('ij, jk -> ik',A,B)

이건 수행불가능한데 $\sum_{j=1}^{n} b_{ij}a_{jk}$ 꼴로 연산을 정의할 수 없기 때문임. ($b$의 두번쨰축과 $a$의 첫번째축의 차원이 같지않음, b의 두번째축은 4, a의 첫번째 축은 2)

np.einsum은 다양한 텐서연산으로 확장할 수 있음. 그런데 @는 2차원 매트릭스만 특화된 연산자임

고차원일 경우에는 @ 를 쓰는게 아님

2차원이하일 경우는 유연하게 해석하여 처리하는데 3차원 이상의 경우 맨 마지막 2개의 차원만 인정함 (저도 이건 몰랐어ㅛㅇ.. 저는 처음 2개의 차원으로 알고있었어요)

헷갈리니까 2차원 넘어가면 np.einsum을 써야함
