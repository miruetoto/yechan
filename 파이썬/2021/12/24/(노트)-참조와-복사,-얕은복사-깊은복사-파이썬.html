참조와 에일리어싱

- 아래의 코드를 관찰하자.

a=[1,2,3] 
b=a 
a=a+[4]

현재 a의 출력결과는?

a

[1, 2, 3, 4]

현재 b의 출력결과는?

b

[1, 2, 3]

당연하다.

- 이제 다시 아래의 코드를 관찰하자.

a=[1,2,3]
b=a
a.append(4)

현재 a의 출력결과는?

a

[1, 2, 3, 4]

현재 b의 출력결과는?

b

[1, 2, 3, 4]

이거 왜이러지??

b=[1,2,3]

아니야?

- 다시 아래의 코드를 살펴보자.

a=[1,2,3]
b=a
a.append(4)

a,b라는 변수들은 메모리에 어떻게 저장되어있을까?

상상력을 조금 발휘하면 아래와 같이 여길 수 있다.

(1) 메모리는 방이 여러개 있는호텔이라고 생각하자. (방은 변수를 저장할 수 있는 공간이다)

(2) 아래를 실행했을 때

a=[1,2,3]
  • 메모리주소1에 존재하는 방을 앞으로 a라고 부르자. 그리고 그 방에 [1,2,3]을 넣는다.

(3) 아래를 실행했을 때

b=a
  • 메모리주소2에 존재하는 방을 b라고 부르고 그 방에 a를 넣는다.
  • 그런데 a=[1,2,3]이므로 b라는 방 역시 [1,2,3]이 있다.

(4) 아래를 실행했을 때

a.append(4)
  • 방 a로 가서 [1,2,3]을 [1,2,3,4]로 바꾼다.
  • 방 b는 아무일도 일어나지 않는다.

- R에서는 맞는 비유인데 파이썬에서는 비유이다. 왜?

id(a) ## 변수 a가 저장된 메모리주소(=방번호)를 출력하는 함수 

140628239102272
id(b) ## 변수 b가 저장된 메모리주소(=방번호)를 출력하는 함수 

140628239102272

실제로는 변수 a,b의 방주소가 동일하기 때문이다.

- 따라서 파이썬에서는 아래와 같은 비유가 더 적절하다.

(1) 메모리는 방이 여러개 있는호텔이라고 생각하자. (방은 변수를 저장할 수 있는 공간이다)

(2) 아래를 실행했을 때

a=[1,2,3]
  • 메모리주소1(=140628239102272)에서 [1,2,3]을 만든다.
  • 메모리주소1(=140628239102272)의 방문에 a라는 포스트잇을 붙인다.
  • 따라서 앞으로 [1,2,3]을 찾으려면 a라고 적힌 포스트잇을 찾으면 된다.

(3) 아래를 실행했을 때

b=a
  • b라는 포스트잇을 새로 만들고 그 포스트잇은 a와 같은 효과를 주도록 한다.
  • 쉽게말하면 b라는 포스트잇을 만들고 메모리주소1(=140628239102272)의 방문에 b라는 포스트잇을 붙인다는 의미이다.
  • 따라서 [1,2,3]을 찾으려면 a라는 포스트잇을 찾거나 b라는 포스트잇을 찾으면 된다.

(4) 아래를 실행했을 때

a.append(4)
  • a라는 포스트잇이 붙어있는 방으로 찾아가서 [1,2,3]을 찾고 거기에서 append함수를 써서 [1,2,3,4]로 바꾼다.
  • 같은방에 a,b라는 포스트잇이 붙어이므로 b라는 포스트잇을 찾아가서 내용을 열어보면 [1,2,3,4]가 나온다.

할당문(=)의 이해

- 파이썬에서 할당문을 이해하려면 언제나 오른쪽을 먼저 읽어야 한다.

  • 할당문의 오른쪽에서 객체를 생성하거나 가져온다.
  • 그 후에 레이블을 붙이듯이 할당문 왼쪽의 변수가 객체에 바인딩 된다. (이걸 참조라고 부름)

- b=a는

나는 이미 a가 의미하는걸 알고 있어, 그런데 a가 의미하는걸 b라고도 부르고싶어

라는 것과 같다. 즉 이미 a라고 부르고 있는것을 가져와서 b라고도 부르고 싶다는 의미인데, 이러한 관점에서 b는 a의 별칭(alias)이라고 볼 수 있다.

- 반대로 생각하면 a역시 b의 별명이라고 볼 수 있다.

- 하나의 메모리 주소에 변수의 이름을 바인드 하는 것을 참조라고 하고, 하나의 메모리 주소에 여러개의 변수이름을 바인드 하는 것을 aliasing이라고 한다.

[3, [66, 55, 44]]
l1[1].append(33)

- 예제5: 아래코드의 결과를 관찰하라. 결과가 예제1과 같은 예제2와 같은가?

l1 = [3,[66,55,44]]
l2 = l1.copy()
l1[1] += [33]

Note: += 연산자는 .append()와 동일한 효과이다.

- 예제6: 아래코드의 결과를 관찰하라. 왜 예제5와 다른결과가 나오는가?

l1 = [3,(66,55,44)]
l2 = l1.copy()
l1[1] += (33,)

(해설)

  • 메모리공간에는 -5~255가 저장
  • 추가로 (66,55,44)이 저장될 공간이 필요해짐. 이 공간은 공간X1라고 하자.
  • 추가로 [3,공간X1의값] 이 저장될 공간이 필요해짐. 이 공간은 l1이라 하자.
  • 추가로 공간 l2를 만들고 [3,공간X1의값]을 저장.
  • 공간X1의 값에 33을 추가하고 싶음. 그런데 공간X1에 저장된 값은 튜플인데 튜플은 불변형임
    • $\to$ 값을 못바꿈
    • $\to$ 할수없이 새로운공간 X2를 만들고 거기에 (66,55,44,33)을 넣고 그것을 l2에 맵핑함.

결국 아래와 같은 상황이 되었음

  • 공간X1: (66,55,44)
  • 공간X2: (66,55,44,33)
  • 공간l1: [3,공간X1의값]
  • 공간l2: [3,공간X2의값] (기존에는 [3,공간X1의값이었음])

기억할 것

- 파이썬은 메모리를 아끼기 위해서 shallowcopy 라는 이상한 행동을 한다.

- 오묘하게도 [1,2,3,4,5]와 같이 0차원 자료형으로만 구성된 리스트라면 문제가 되지 않는다. (메모리는 아끼면서 문제가 되지 않는다? 천재인데?)

- 그런데 [[1,2],[3,4]]와 같이 리스트에 리스트가 들어간 형태라면 문제가 생긴다. (이건 개선이 필요하겠는걸?)

  • 개선1: 깊은복사
  • 개선2: numpy

deep copy

- 얕은복사가 아니라 내부객체까지 모두 새로 만드는 깊은복사를 하기 위해서는 아래와 같이 copy 모듈을 사용한다.

import copy

- 깊은복사 예제1

l1 = [3,[66,55,44]]

l2 = copy.deepcopy(l1)

l2[1].append(55)

l1,l2

([3, [66, 55, 44]], [3, [66, 55, 44, 55]])

- 깊은복사 예제2

l1 = [3,[66,[55,44]]]

l2 = copy.deepcopy(l1)

id(l1),id(l2) ## shallow copy = 1 level deep copy 

(140628103869696, 140628103867200)
id(l1[1]),id(l2[1]) ## deep copy = 2 level deep copy 

(140628104264832, 140628105151872)
id(l1[1][1]),id(l2[1][1]) ## deep copy = 3 level deep copy 

(140628104231936, 140628104258176)
l2[1][1].append(33)

l1,l2

([3, [66, [55, 44]]], [3, [66, [55, 44, 33]]])
