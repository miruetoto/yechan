파이썬이 어려웠던 이유

- 파이썬 배우는 초보자에게 가장 어려운것!

  • 선생님마다 설치하는 방법이 모두 다름

- 왜 저렇게 설치방법이 다른가? 왜 다른 방법으로 각각 파이썬을 실행하는가? 이런것이 너무 어려움

  • 방법1: 파이썬프로그램 다운로드 -> 시작버튼 눌러서 설치
  • 방법2: 아나콘다 설치 (그럼 자동으로 파이썬이 설치됨)
  • 방법3: 아나콘다 설치 + 가상환경
  • ...

- 심지어 실행하는것도 다름

  • 방법1: 파이썬 프롬프트
  • 방법2: .py를 이용하여 실행?
  • 방법2: IDLE
  • 방법3: 아나콘다 네비게이터
  • 방법4: 주피터
    • 가상환경을 만들어서 해라..
    • 아나콘다 네비게이터에 주피터가 있다..
  • 방법5: 파이참
  • 방법6: 스파이더
  • 방법7: Visual Studio Code
  • ...

- 심지어 버전도 다름

  • 어떤책은 파이썬 3.7을 기반으로 코드작성했고 그 외의 파이썬에서는 실행되지 않을수도 있다고 위협을 가함..

- 머리아프니까 collab을 쓰라는 사람도 있음. 아니면 도커이미지를 줄테니까 그걸 쓰라는 사람도 있음. AWS를 쓰라는 사람도 있음.. $\to$ 이게 더 머리아픔

- 핵심: 그냥 (1) 컴퓨터에 (2) 파이썬을 깔아서 (3) 실행하는것임

- 의문: 그럼 왜 엑셀처럼 프로그램설치하고 아이콘 더블클릭하면 끝나는 식으로 만들지 않는가? 왜 이렇게 힘들어야 하는가?

개발환경 구축방법이 많은 이유?

- 파이썬 개발환경 구축은 수많은 방법이 있다. (검색하면 진짜 무수히 많이 나옴)

- 이는 마치 라면의 레시피를 검색했을때 수많은 방법이 나오는것과 유사함.

  • 방법1: 스프를 먼저 넣고 끓인다음에 라면을 넣어야 합니다.
  • 방법2: 양은냄비에 물넣고 물 끓으면 스프넣고 라면 넣고 마지막에 계란을 넣는다.
  • 방법3: 웤에 물을 조금 넣고 라면과 스프를 넣고 자글자글 끓이다가 마지막에 참기름을 넣어요
  • 방법4: 먹다남은 삼겹살을 후라이팬에 볶은뒤에 물을 붓고 라면을 먹는다.
  • 방법5: 용기에 라면과 스프와 뜨거운물을 넣고 랩을 쒸운뒤에 젓가락으로 구멍을 뚫는다. 그 다음에 전자렌지로 돌린다.
  • ...

- 우리는 모든 방법을 나열할 순 없지만 모든방법을 이해할 수 있다. 왜냐하면 라면을 끓이는 어떤한 공통맥락을 우리는 파악하고 있기 때문.

- 파이썬의 개발환경 구축 역시 사실 비슷하다. 공통맥락을 파악하면 쉽다.

- 제 목적: 가장 맛있는 라면을 끓이는 하나의 레시피가 아니라 라면을 끓이는 레시피를 이해할 수 있는 공통맥락을 설명하고 싶어요

Note: 설치하는 방법이 다양한 이유? 파이썬이 인기있어서 + 다양한 방법으로 설치하면 각각의 장점이 뚜렷해서 (뻔한건 다양하지 않아요.. 감자삶는 레시피가 다양하진 않아요..)

3세대 프로그래머: 메모장 + anaconda prompt + python / 메모장 + ipython

방법1: 메모장 + anaconda prompt + python

- 전체적인 수행방법은 아래와 같다.

  • 메모장: 코드를 편집 수정
  • anaconda prompt: 메모장에 저장된 코드를 실행하고 결과를 확인
  • python: 코드의 결과를 그때그때 확인하며 짧은 단위의 코드를 만듬

- 방법1의 아쉬운점

  • 아쉬움1: 메모장, anaconda prompt, python 3개를 동시에 사용하는 것이 난잡하다.
  • 아쉬움2: ananconda prompt에서 "python myadd.py"를 실행해도 실행내용이 python에 공유되지 않는다. (독립적인 두개의 python이 있는것임)

- 상황극: 1~10까지의 수를 더함에 있어서 아래의 코드가 동작하지 않았다고 가정하자.

### myadd.py 
total=0
for i in range(0,10):
    total = total +i 
    i=i+1 
print(total)

코드의 문제점을 살펴보다가 혹시 마지막순간에 i=10이 아니라 i=9가 들어가 있는것은 아닐까? 하는 의심이 들어서 i값을 확인해보고 싶어졌다.

(시도1) python 콘솔에 i를 쳐본다. (이게 동작하지 않는다!)

(시도2) myadd.py의 마지막줄을 print(i)로 수정하여 확인한다.

방법2: 메모장 + ipython

- 이 방법은 메모장 + anaconda prompt + python 조합보다 단순하면서 더 많은 기능을 제공한다.

  • 메모장: 코드를 편집 수정
  • ipython: 코드의 결과를 그때그때 확인하여 짧은 단위의 코드를 만듬 + anaconda prompt의 역할은 !python myadd.py를 통하여 수행가능

    추가기능:%run myadd.py를 이용하여 현재 프롬프트내에서 myadd.py의 내용을 그대로 실행할 수 있다. ($\star$)

- 난잡하지도 않고 (아쉬움1을 해결) 경우에 따라서는 myadd.py의 실행내용 및 결과를 그대로 ipython에서 확인할 수 있다.

4세대 프로그래머: IDE의 등장

- 메모장+ipython을 하나로 통합한 프로그램들이 등장!

  • jupyter notebook, jupyter lab
  • syder
  • idle
  • VScode

- 주피터의 트릭 (실제로 주피터는 ipython에 기생할 뿐 아무것도 안해요)

(1) 주피터가 실행

(2) 새 노트북 생성 (파이썬으로 선택)

  • 컴퓨터는 내부적으로 ipython 커널이 실행됨 (증거: 오른쪽 상단)

(3) 처음 보이는 cell에 1+1 입력 쉬프트+엔터를 누름 -> 결과 2 출력

  • 처음 보이는 cell 하나 = 자동으로 열린 메모장 하나
  • cell에 1+1 입력 = 메모장에 1+1 적음
  • 쉬프트+엔터후 결과2출력 = cell의 내용을 복사 -> ipython에 붙어넣기 -> ipython 결과를 복사 -> cell로 돌아와 붙여넣기

(4) 새로운 cell을 추가하고 2+2 입력 쉬프트+엔터 누름 -> 결과 4출력

  • 새로운 cell 추가 = 새로운 메모장 추가
  • cell 에 2+2 입력 = 새로운 메모장에 2+2 적음
  • 쉬프트+엔터후 결과4출력 = cell의 내용을 복사 -> ipython에 붙어넣기 -> ipython 결과를 복사 -> cell로 돌아와 붙여넣기

- 중요한 사실들

  • IDE는 내부적으로 연산수행능력이 없다. (생각해볼것: 왜 R을 꼭 설치하고 Rstudio를 설치해야 했을까?)
  • 주피터에서 커널을 재시작한다는 의미는 메모장은 모두 열려있는채로 ipython을 껐다가 다시 실행한다는 의미이다.
  • 주피터는 단순히 메모장내용을 복붙하여 전달하는 기계라고 생각할 수 있다. 이렇게 생각하면 주피터에 꼭 ipython을 연결할 이유는 없다. 실제로 주피터에 R을 연결하여 쓸 수 있다. 즉 하나의 IDE가 여러개의 언어와 연결될 수 있다.
  • jupyterlab이라는 프로그램은 크롬에 있는 내용과 ipython간의 통신을 제어하는 프로그램일 뿐이다.
  • 통신제어상태는 jupyter lab을 치면 유지되는 ananconda prompt에서 확인가능하다.
  • anaconda prompt를 끄면 통신을 담당하는 jupyter lab이 꺼지는 것과 동일하므로 크롬은 그냥 크롬이 되어버린다.

5세대 프로그래머: 가상컴퓨터(anaconda), 원격컴퓨터(server), 클라우드컴퓨터의 개념 등장

- 생각해보니까 어차피 ipython이 실행된 컴퓨터에 내가 크롬에서 입력한 명령이 "전달"되기만 하므로 꼭 같은 컴퓨터일 필요는 없음.

- 모델1: 원격컴퓨터

(준비상태)

  • 전북대컴퓨터: ipython 실행 + 이 컴퓨터는 인터넷 연결이 되어있음
  • 우리집노트북: 크롬실행 + 이 컴퓨터는 인터넷 연결이 되어있음

(명령입력)

  • 우리집노트북에서 1+1을 크롬에 입력하고 쉬프트 엔터를 누름

(우리집노트북->전북대컴퓨터)

  • 우리집노트북 내부의 어떤 프로그램은 1+1이라는 명령을 복사하여 카카오톡으로 전북대컴퓨터에 전달
  • 전북대컴퓨터 내부의 어떤 프로그램은 1+1이라는 명령을 카톡으로 받아서 그것을 ipython에게 전달

(전북대컴퓨터->우리집노트북)

  • 전북대컴퓨터 내부의 ipython은 2라는 출력결과를 계산함
  • 전북대컴퓨터 내부의 어떤 프로그램은 계산결과를 카톡으로 우리집 노트북에게 전달
  • 나는 우리집 노트북에서 계산결과를 받아볼 수 있다.

- 모델2: 원격컴퓨터 + 가상컴퓨터

(준비상태)

  • 성능 좋은 전북대 컴퓨터1개
  • 내 노트북1개 (그냥 싸고 가벼운거)
  • 대학원생 노트북1개 (그냥 싸고 가벼운거)

(아이디어)

  • 성능좋은 컴퓨터를 논리적으로 3개로 나눈다. (물리적으로 뿌개는것이 아니라 논리적으로) 이를 각각 (base), (py39jl17), (py38r40) 컴퓨터라고 하자.
  • 나는 (py39jl17)에 접속하여 파이썬 3.9와 줄리아 1.7을 설치한뒤 실습한다.
  • 대학원생은 (py38r40)에 접속하여 파이썬 3.8과 R 4.0을 설치한뒤 실습한다.
  • (base)는 예비용으로 아무것도 설치안한 깨끗한 상태유지
  • 내가 뭘 실수하여 (py39jl17)컴퓨터가 망가졌으나 (py38r40)은 아무런 타격없다.
  • 나는 (py39jl17)를 삭제하고 (base)로 부터 다시 새로운 컴퓨터 (py39jl17new)를 만들어 사용한다.

- 모델3: 가상컴퓨터 (여러분사례)

  • 여러분의 컴퓨터를 (base), (py39) 2개의 컴퓨터로 나눔
  • 여러분들은 (py39)에만 주피터랩 설치
  • (py39)에 있는 ipython과 여러분의 크롬창이 서로 교신하면서 실습
  • 장점: 서로 다른 환경에 서로다른 파이썬과 R등을 설치할 수 있다. $\to$ 패키지간 충돌 최소화 (파이썬 입문 수업듣고 시각화 수업들었는데 거기서 머 잘못되어서 갑자기 파이썬입문 강의노트까지 망가짐)

- 모델4: 클라우드

(사례1) 사실 일반이 구축하기에는 기술적으로 거의 불가능

  • 성능이 그저그런 컴퓨터 3개
  • 대학원생 포함 쓸 사람은 5명
  • 3대의 컴퓨터를 논리적으로 5대로 할당하고 5명이서 사용

(사례2) 구글코랩

  • 구글에 여러성능을 가진 컴퓨터가 $n$대 있음
  • $m$명의 사람이 $n$대의 컴퓨터에 접속
  • 적당히 컴퓨터 자원의 분배하여 나눠서 씀

요약 및 정리

- 주피터랩에서 아래 명령이 가지는 의미를 기억하자.

!python myadd.py 
%run myadd.py

- 결국 (1) 컴퓨터에 (2) 파이썬을 설치하고 (3) 실행하는 과정은 생각보다 다양한 선택의 조합이 있다.

(1) 그냥 내노트북에 파이썬을 설치할지? 내 노트북안에 가상컴퓨터를 만들고 거기에 파이썬을 설치할지? 아니면 학교의 데스크탑에 파이썬을 설치하고 접속하여 쓸지? 설치안하고 구글컴퓨터에 설치된 파이썬을 난 쓰기만 할지?

(2) python을 설치할지? ipython을 설치할지? 아니면 어차피 가상환경 나중에 쓸꺼니까 anaconda로 설치할지? 아니면 코랩쓸꺼니까 설치 안할지?

(3) 어떠한 IDE를 쓸지? IDE를 쓰지 않을지? 내가 IDE를 직접구성하여 쓸지?

- 하지만 공통적인으로 관통하는 원리가 있다.
