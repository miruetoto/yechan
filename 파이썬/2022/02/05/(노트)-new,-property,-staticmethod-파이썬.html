__new__

- __new__: 사용자가 클래스()를 입력하였을 경우 동작하는 명령을 정의한다. // __new__를 따로 정의하지 않아도 자동으로 어떤 동작이 정의되어있는데 그 동작은 '클래스->인스턴스화'.

class Report: 
    def __init__(self):

class ReportCover: 
    def __new__(cls):
        return super().__new__(cls)
    def __init__(self):
        ### 추가코드1
        ### 추가코드2        
        self.a= '과목명: 파이썬입문'

실행순서에서 차이점이있긴있음

  • 코드1: 추가코드1 -> 추가코드2 -> 인스턴스생성 -> self.a= '과목명: 파이썬입문' -> self.a= '과목명: 파이썬입문'
  • 코드2: 인스턴스생성 -> 추가코드1 -> 추가코드2 -> self.a= '과목명: 파이썬입문' -> self.a= '과목명: 파이썬입문'

- 엄청특별한 경우 아니면 __new__ 로 구현가능한 기능은 모두 __init__ 를 통해서도 구현가능 함

__init__으로는 구현불가능하지만 __new__로는 구현 가능한 것

예제2는 의미가 좀 이상한데 양식문서의 제목을 바꾸기 위해서는 인스턴스를 만든뒤에 메소드를 적용해야함.

이 아니라

Report.change('과목명: 데이터시각화')

class Report: 
    a= '과목명: 파이썬입문'
    def __init__(self):
        self.a=Report.a    
    @staticmethod
    def change(newname):
        Report.a=newname

report=Report()
report.a

'과목명: 파이썬입문'
Report.change('과목명: 데이터시각화') ## 클래스의 제목변경 
report.a ## 인스턴스에는 영향없음. 

'과목명: 파이썬입문'
report2=Report() ## report2부터는 바뀐제목이 적용
report2.a 

'과목명: 데이터시각화'
