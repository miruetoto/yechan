복사

참조는 이제 지긋지긋하다. 동일한 내용을 다른메모리주소에 넣자. (포스트잇같은거 하지말고 방의 개념으로 하자고)

- 예제1

(코드1)

l1 = [3,[66,55,44]]
l2 = l1 ## 참조 
l1,l2

([3, [66, 55, 44]], [3, [66, 55, 44]])
id(l1),id(l2)

(139636477568640, 139636477568640)
  • 주소가 같다. 
l1[0]=4 # 에일리어싱개념없는사람: l1의 원소를 바꿨는데 왜 l2도 같이바뀌나..

l1, l2 

([4, [66, 55, 44]], [4, [66, 55, 44]])
l2.append(5)

l1,l2 # 에일리어싱개념없는사람: l2를 바꿨는데 왜 l1 도 같이바뀌나.. 

([4, [66, 55, 44], 5], [4, [66, 55, 44], 5])
  • 정말 짜증난다. 이제 여기서 벗어나자.

(코드2)

l1 = [3,[66,55,44]]

l2 = list(l1) ## 카피

id(l1),id(l2) ## 주소가 다르다!!

(139636477523456, 139636477054784)
l1[0]=4

l1,l2

([4, [66, 55, 44]], [3, [66, 55, 44]])
  • 이제 상식적인 프로그래밍언어가 되었음

그런데..

깊은복사 

import copy

- 예제1

l1 = [3,[66,55,44]]

l2 = copy.deepcopy(l1)

l2[1].append(55)

l1,l2

([3, [66, 55, 44]], [3, [66, 55, 44, 55]])

(해설)

l1 = [3,[66,55,44]]

l2 = copy.deepcopy(l1)

139636476751168
